# Single vs Multi Agent

Single agents are simpler but suffer from tool overload, context overload, and single-point failure. Multi-agent systems isolate roles, tool scopes, and context, but require explicit orchestration.

## Choose Multi-Agent When

- The task has distinct specialist roles.
- Different tools should be isolated by responsibility.
- Work can run in parallel.
- Failures need independent retry or partial-result preservation.
