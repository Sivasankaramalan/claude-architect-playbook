# Glossary

Shared vocabulary for the **Claude Certified Developer – Foundations (CCDV F)** guide. Terms reflect public Anthropic API, agent, MCP, and Claude Code concepts—verify exact behavior in official docs before exam day.

---

## A

**Agent loop** — Control flow that repeatedly calls the Messages API until `stop_reason` is `end_turn` or a limit is hit; executes tools when `stop_reason` is `tool_use`.

**Agent SDK** — Anthropic SDK layer for building agents with sessions, hooks, subagents, and tracing beyond raw Messages API calls.

**Anthropic API** — HTTP API for Claude models, primarily the Messages API for conversational and tool-using applications.

---

## B

**Batch (Message Batches)** — Asynchronous API for processing many Messages requests offline with separate latency and pricing from synchronous calls.

---

## C

**Cache breakpoint** — Position in the request prefix where prompt caching starts; content before the breakpoint must be identical across requests to get cache reads.

**Cache read tokens** — Input tokens billed at reduced rate when a matching cached prefix is reused.

**Cache write tokens** — Tokens written to cache on first use of a cacheable prefix (or after invalidation).

**CLAUDE.md** — Project-level instructions file read by Claude Code; part of the configuration hierarchy for team coding agents.

**Context window** — Maximum tokens processable in one request, including system prompt, tools, history, and attachments.

**Context engineering** — Designing what information enters each model call: retrieval, pruning, summarization, and external memory.

---

## D

**Deterministic enforcement** — Guarantees via code (schemas, hooks, validation, permissions) rather than prompt instructions alone.

---

## E

**End turn (`end_turn`)** — `stop_reason` value indicating the model finished its turn without requesting more tools; agent loop should stop or validate output.

**Eval (evaluation)** — Structured test of model/application behavior using golden cases and graders (code, human, or model-judge).

**Extended thinking** — Mode where the model uses additional internal reasoning tokens before answering; higher quality/latency/cost tradeoff.

---

## F

**Few-shot prompting** — Including example input/output pairs in the prompt to steer format and behavior.

**Fail closed** — Security/design pattern where errors or validation failures deny action rather than proceeding with reduced checks.

---

## G

**Guardrail** — Programmatic constraint on inputs, outputs, or tool calls (schema validation, hooks, classifiers, approval gates).

---

## H

**Haiku** — Fast, cost-efficient Claude model tier suited for routing, classification, and high-volume simple tasks.

**Hook** — User-defined callback invoked at agent lifecycle events (e.g., before tool execution) for logging, blocking, or transformation.

**Human-in-the-loop (HITL)** — Workflow where a person approves high-risk actions before tools execute.

---

## I

**Indirect prompt injection** — Adversarial instructions embedded in untrusted content (documents, emails, web pages, tool results) rather than the user message directly.

**Input tokens** — Tokens sent to the model: system, messages, tools, images, documents.

---

## M

**MCP (Model Context Protocol)** — Open protocol for connecting Claude to external tools, resources, and prompts via MCP servers.

**Max tokens** — API parameter capping output length; can cause `stop_reason: max_tokens` if exhausted.

**Message Batches** — See **Batch**.

**Messages API** — Primary Claude API for multi-turn conversations, tool use, vision, PDF, streaming, and prompt caching.

**Model Context Protocol** — See **MCP**.

---

## O

**Opus** — Highest-capability Claude model tier for complex reasoning and high-stakes synthesis; slower and costlier.

**Orchestration** — Application control logic for agent loops, routing, retries, and workflow state—owned by your code, not the model.

**Output tokens** — Tokens generated by the model in its response, including visible text and structured tool calls.

---

## P

**Parallelization** — Running independent tool calls or worker agents concurrently to reduce wall-clock time.

**Prompt caching** — API feature reusing precomputed prefix context across requests to lower latency and input cost.

**Prompt injection** — Attack where untrusted text attempts to override system instructions or exfiltrate data.

**Pruning** — Removing or compressing conversation history and tool outputs to save tokens and improve signal-to-noise.

---

## R

**RAG (retrieval-augmented generation)** — Fetching relevant documents at query time and injecting them into context instead of sending entire corpora.

**Rate limit (429)** — API response when request volume exceeds tier limits; requires backoff and concurrency control.

**Reflection** — Pattern where a draft output is critiqued or verified in a subsequent pass before delivery.

**Request ID** — Unique identifier in Anthropic API response headers for support and log correlation.

**Resume / continue** — Claude Code session flags to restore prior conversation context vs starting fresh.

**Routing** — Sending requests to appropriate models, prompts, or tool subsets based on intent or rules.

---

## S

**Sandbox** — Isolated environment for code or tool execution with restricted network, filesystem, and credentials.

**Session** — Persistent conversation state keyed by identifier; used by SDK and Claude Code for multi-turn work.

**Skills** — Packaged instruction modules for Claude Code extending agent capabilities in defined domains.

**Sonnet** — Balanced Claude model tier; common default for coding agents and production assistants.

**Stop reason (`stop_reason`)** — Response field indicating why generation stopped: e.g., `end_turn`, `tool_use`, `max_tokens`.

**Streaming** — Receiving model output incrementally via SSE events; improves time-to-first-token for interactive UX.

**Structured output** — Constraining model responses to JSON Schema or tool schemas for reliable parsing.

**Subagent** — Child agent run with isolated context delegated by a coordinator; does not automatically inherit full parent history.

**Supervisor pattern** — Coordinator agent delegates to workers and synthesizes results.

**System prompt** — Developer-provided instructions establishing role, constraints, and behavior—not user messages.

---

## T

**Thinking tokens** — Tokens used in extended thinking modes; billed and counted separately from visible output in usage metrics.

**Tool choice (`tool_choice`)** — API parameter controlling whether/which tool the model must call (`auto`, `any`, `tool`, or specific tool).

**Tool result** — Message role carrying execution output linked to a prior `tool_use` block via `tool_use_id`.

**Tool use (`tool_use`)** — Content block where the model requests invocation of a defined tool with structured input.

**Tracing** — Distributed spans tracking a request across API calls, tools, and MCP for latency and debugging.

**TTFT (time to first token)** — Latency until the first streamed output chunk arrives.

---

## V

**Validation** — Server-side or post-model checks that outputs/tools meet schemas and business rules before proceeding.

---

## X

**XML tags** — Delimiters (e.g., `<document>`, `<instructions>`) structuring prompts and separating trusted from untrusted content.
