# Contributing

Thanks for helping improve the Claude Certified Developer community guide.

## What To Contribute

- Original explanations and diagrams
- High-level summaries of **public** Anthropic docs
- Original mock questions and practice prompts
- Fixes for typos, broken links, or outdated public-API notes
- Clear examples under `examples/`

## What Not To Contribute

- Real exam questions or dumps
- Private employer / client training material
- Proprietary PDFs, slides, videos, or copied course content
- Credentials, private URLs, emails, or tokens

## How To Contribute

1. Open an issue describing the gap or fix.
2. Keep changes inside `developer/` unless the change is repo-wide.
3. Prefer small PRs: one topic or one section at a time.
4. Match existing markdown style (short sections, concrete examples).
5. Confirm the publication checklist before opening a PR.

## Style Notes

- Write for developers preparing for a certification exam.
- Prefer “why this matters on the exam / in production” over long theory.
- Link official public docs when citing product behavior.
- Mark stubs clearly if a page is incomplete.

See [`../PUBLICATION_CHECKLIST.md`](../PUBLICATION_CHECKLIST.md).
